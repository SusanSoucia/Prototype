# 2026-06-30 Chat 前端三个问题修复行动报告

## 修改概述

修复前端 Chat 页面三个问题：
1. 新建对话后发送的消息不显示（mock 环境）
2. 文档预览功能缺少 mock 数据支持
3. 流式传输时滚动条异常反复回顶部

共修改 **5 个文件**，所有修改已通过 TypeScript 类型检查（src/ 零错误）。

---

## 问题 1：新建对话不显示发送的消息 ✅

### 根因

`chat-list.vue` 的 `watchEffect(() => getList())` 在 `conversationId` 变化时异步调用 `GET /users/conversation`，返回后用 `list.value = data` 整体替换消息列表。在 mock 环境下新 conversationId 无数据，返回 `[]` 覆盖掉用户刚 push 的本地消息。

同时 store 的 `createNewSession()` / `switchSession()` 也调用 `loadMessages()`，形成双重加载路径，增加了竞态窗口。

### 修改

| 文件 | 内容 |
|------|------|
| `frontend/src/store/modules/chat/index.ts` | (1) `createNewSession()` 移除 `list.value = []` 和 `await loadMessages()` — 只设 `conversationId`，消息加载交由组件 watcher；(2) `switchSession()` 同样移除；(3) `loadMessages()` 增加 CID 守卫：`if (cid !== conversationId.value) return` |
| `frontend/src/views/chats/modules/chat-list.vue` | (1) 替换 `watchEffect(() => getList())` 为显式 `watch([conversationId, range], () => getList())`；(2) `getList()` 增加 `fetchVersion` 计数器和 `capturedCid` 守卫，丢弃过期异步响应；(3) 增加 `hasInFlight` 守卫：如果本地有 pending/loading 状态的 assistant 消息且服务器返回空，保留本地消息不覆盖 |

## 问题 2：文档预览功能 mock 数据 ✅

### 根因

`chat-message.vue` 和 `reference-preview.vue` 点击来源链接时会请求 `GET /documents/reference-detail`，但 mock 中不存在此端点。此外 mock 数据格式存在多处不一致：

- `referenceMappings` 在 `build/plugins/mock.ts` 的 WS completion 事件中使用数组格式，类型定义要求对象格式
- `chunkId` 为字符串，类型定义要求 number
- `retrievalMode` 使用 'semantic'/'keyword'，类型定义要求 'HYBRID'/'TEXT_ONLY'

### 修改

| 文件 | 内容 |
|------|------|
| `frontend/build/plugins/mock.ts` | (1) 新增 `GET /documents/reference-detail` 处理器，返回符合 `Api.Document.ReferenceDetailResponse` 类型的 mock 数据（含 fileMd5、fileName、pageNumber、anchorText、retrievalMode、matchedChunkText、evidenceSnippet、score、chunkId 等字段）；(2) 新增 `POST /chat/feedback` 处理器；(3) WebSocket completion 事件 `referenceMappings` 从数组改为对象格式（key 为 "1"/"2"）；(4) `chunkId` 从字符串改为数字；(5) `retrievalMode` 统一为 'HYBRID' / 'TEXT_ONLY'；(6) mock 消息数据的 `referenceMappings` 同步修正 |
| `frontend/mock/chat.ts` | (1) 新增 `GET /documents/reference-detail` 路由条目；(2) 新增 `POST /chat/feedback` 路由条目；(3) `fileMd5` 值统一为 32 位十六进制字符串 |

## 问题 3：流式传输滚动条异常 ✅

### 根因

三个因素叠加：
1. **问题 1 连锁反应**：`getList()` 覆盖 `list.value` → Vue 重建 DOM → 滚动位置回到顶部
2. **过度滚动调用**：`watch(() => [...list.value], scrollToBottom)` 在每条 chunk 到达时触发，`setTimeout(100ms)` + `scrollBy({ top: 999999999999999 })` 造成频繁抖动
3. **computed 副作用**：`chat-message.vue` 的 `content` computed 中调用 `chatStore.scrollToBottom?.()`，每次渲染都触发额外滚动

### 修改

| 文件 | 内容 |
|------|------|
| `frontend/src/views/chats/modules/chat-list.vue` | (1) 移除 `watch(() => [...list.value], scrollToBottom)`；(2) 实现智能滚动：使用 `watchEffect` 跟踪消息数量和内容长度，区分"新消息"（smooth 滚动）和"流式更新"（instant 滚动）；(3) 新增 `userScrolledUp` 状态：用户手动上滚超过 80px 时停止自动跟随；(4) 用户发送新消息时自动恢复自动滚动；(5) 切换 conversation 时重置滚动状态；(6) 使用 `nextTick` 替代 `setTimeout`；(7) `<NScrollbar>` 绑定 `@scroll="onScroll"` |
| `frontend/src/views/chats/modules/chat-message.vue` | 移除 `content` computed 中的 `chatStore.scrollToBottom?.()` 副作用调用 |

## 验证方式

1. **问题 1**：登录 → 点击"新对话" → 输入消息并发送 → 确认用户消息和 AI 流式回复均正常显示
2. **问题 2**：在已有对话中点击 AI 回复中的来源链接 → 确认新页面正确展示文档引用详情（文件名、页码、锚文本、检索模式、评分等）
3. **问题 3**：发送消息 → 观察流式传输时滚动条稳定跟随内容底部 → 手动上滚后不被拉回 → 再次发送消息恢复自动跟随
